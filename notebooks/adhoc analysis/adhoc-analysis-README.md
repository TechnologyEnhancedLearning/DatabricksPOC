# Whats in the folder

This folder is for adhoc requests for data, ussually quick discoveries with data, outputs and recommendations copied into excel.

We want to keep a copy of these for future in case of overlapping requests.

Providing sufficient information in these folders will enable us to find, or use AI to find similar previous tasks.

Recording limitations, observations for future will enable us to benefit from future learning.

# Folder Naming and Contents

Folders need to be easily discoverable and have enough information for context.

Folders will be named: TD-<jira ticket number>-MMyy-<description>
With the aim to make them findable by ticket, theme or date, the ticket numbers will generally be chronological so date will be roughly chronological within the folder contents order.

## Contents
Each folder should contain a notebook which represents all the client facing outputs, to be put into excel for example.

*Remember this is source controlled so additional or sensative information should be shared in the ticket itself, which will be linked in the file*

In addition to client facing notebook will be a readme td-xxxx-readme.md
This will contain useful notes for discovery and for future analysts

# Templates
Use the readme template and the notebook template to keep a standard structure, update the template with the team where it can be improved

# Review
The files should be peer reviewed for the benefit of sharing knowledge, approaches and seeing opportunities to make elements reuseable.

# Tips
- AI can help find previous adhoc work for reference
- When the work is PR'd try and find opportunities for making useful code reuseable as utils function (that are unit tested) and gold layer tables
- if it is a time sensative request it might be expedient to put the outputs into excel before reviewed and published but it may still be worth getting some feedback first
