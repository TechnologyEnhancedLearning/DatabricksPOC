## Great Expectations
Great Expectation data quality package is an additional learning curve and more specific packaging will make AI aid in the learning python less effective.
Stackoverflow shows mixed opinions, which is shared by the NHS databricks community. Tools like fluent assertion look really clean, and pipeline steps for failing before processing data is useful (though likely already acheivable with the current set up). If we want to do this in future we should contact another team that has success with it.

The advantages of quality checking on pipelines rather than merge so not just testing once, for now, can be best achieved with dlt.expect native functionality
